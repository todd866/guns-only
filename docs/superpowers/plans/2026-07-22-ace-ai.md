# Ace AI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the AI a *credible threat* — today a player is never even shot at, because the reactive bandit is coded "deliberately beatable" and never achieves a firing solution against a competent pilot. Give it a skill ladder that can actually pursue, gun, and hit the player, culminating in a smart **ace** (max-performance energy fighting, overshoot-forcing gun defence, tactical disengage/re-engage). Success is measured by **player win%** against each tier, tuned to a reasonable band (`docs/roguelite-loop-design.md`, "The ace AI" and success criteria).

**Architecture:** Extend the existing `ReactiveBandit` (not a new class) with a `PilotSkill` competence profile that gates decision quality and unlocks ace-only behaviours. The default profile reproduces today's "competent" bandit exactly (a regression guarantee), so early gauntlet fights are unchanged and only higher tiers get smarter. All logic stays in `sim/Doctrine/` behind `IBandit`; it consumes only the player state already passed to `Step` (no hidden truth), owns a normal `AircraftSim`, and stays deterministic (no wall-clock, no RNG — variety is keyed on engagement number).

**Tech Stack:** C# (.NET 8), `GunsOnly.Sim` kernel, xUnit (`sim.Tests`). Build/test with the pinned SDK:
`PATH="/opt/homebrew/bin:$PATH" GUNS_DOTNET_CLI="$HOME/.dotnet/dotnet" DOTNET_ROOT="$HOME/.dotnet" DOTNET_MULTILEVEL_LOOKUP=0 ./bin/check`, or a focused run:
`$HOME/.dotnet/dotnet test sim.Tests/GunsOnly.Sim.Tests.csproj --filter "FullyQualifiedName~AceBandit"`.

## Global Constraints

- **Primary success metric = player win%.** The AI must actually threaten. Instrument win% two ways: (1) a headless **reference-player** model of defined competence in the duel harness, measuring what fraction of representative fights the reference player wins vs each AI skill tier; (2) real-sortie win/loss from the existing telemetry recorder, aggregated per opponent tier. Tune the skill tiers so a competent-but-not-perfect player wins roughly **40–70%** against the gauntlet's mix (the ace should be *hard*, not a bullet-sponge; early tiers should still be beatable). The concrete tuned numbers are the deliverable of Task 8; the criterion is "you get shot at and can lose."
- **Determinism.** No `DateTime`/`Stopwatch`/`Environment.TickCount`, no unseeded RNG in any tick. Variety is a pure function of engagement number and geometry. (Kernel invariant; verified by determinism tests.)
- **No renderer/browser types** in `sim/`.
- **No hidden truth / no privileged outcomes.** The AI may read only the `in AircraftState player` handed to `Step`; the session still owns ammunition, projectiles, damage, and outcomes. `WantsToFire` stays routed through `BanditFireControl` (honest gun discipline — same ballistics the player faces).
- **Behaviour-preserving default.** The `PilotSkill.Competent` profile must reproduce the current `ReactiveBandit` output tick-for-tick; existing `ReactiveBanditTests` must stay green unchanged.
- **Float64 authoritative state, RK4 `AircraftSim`, fixed 120 Hz tick** (`Dt = 1.0 / AircraftSim.TickHz`).

```csharp
// sim.Tests/BfmDuel.cs
using GunsOnly.Sim;
using GunsOnly.Sim.Doctrine;

namespace GunsOnly.Sim.Tests;

public readonly record struct BfmDuelResult(
    double ASolutionSeconds, double BSolutionSeconds, double MinRangeM,
    double AExitSpeedMps, double BExitSpeedMps);

public static class BfmDuel {
    const double Dt = 1.0 / AircraftSim.TickHz;
    public static BfmDuelResult Fly(ReactiveBandit a, ReactiveBandit b, double seconds) {
        double aSol = 0.0, bSol = 0.0, minRange = double.PositiveInfinity;
        int ticks = (int)(seconds * AircraftSim.TickHz);
        for (int i = 0; i < ticks; i++) {
            var aState = a.State; var bState = b.State;   // same beginning-of-tick sample for both
            a.Step(bState, Dt);
            b.Step(aState, Dt);
            if (CameraSolver.GunWindow(a.State, b.State)) aSol += Dt;
            if (CameraSolver.GunWindow(b.State, a.State)) bSol += Dt;
            double range = Geometry.Range(a.State, b.State);
            if (range < minRange) minRange = range;
        }
        return new BfmDuelResult(aSol, bSol, minRange, a.State.Speed, b.State.Speed);
    }
}
```

```csharp
// sim/Doctrine/CompetenceLadder.cs
namespace GunsOnly.Sim.Doctrine;

public static class CompetenceLadder {
    public static PilotSkill ForFight(int fightIndex, int fightCount) {
        if (fightCount <= 1) return PilotSkill.Ace;
        double t = (double)fightIndex / (fightCount - 1);   // 0..1 across the run
        return t >= 0.999 ? PilotSkill.Ace
            : t >= 0.66 ? PilotSkill.Veteran
            : t >= 0.33 ? PilotSkill.Competent
            : PilotSkill.Novice;
    }
}
```
